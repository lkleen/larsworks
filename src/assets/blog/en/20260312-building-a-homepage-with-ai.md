During the last year AI has made great steps forward in generating source code. Around 2024 I started to use it on a regular basis.
During that time it has evolved from a tool which could hardly generate compiling code to being as smart that it can now act as a
team of software developers and architects for me.

I'm working as a technical lead since 2018. What I'm doing with AI is basically the same as what I'm doing with my human engineering colleagues.
I'm evaluating frameworks and how they are matching actual requirements and make some considerations how specific decisions
will influence the efficiency of the development process in the future. Normally I'm the person who is than writing specifications
and architecture documentations. It very much depends on the quality of my spec and the maturity of the team how this translates to
actual source code. And it always involves many cycles of writing, reviewing and refactoring code to achieve a good result.

Leveraging AI I can do that on my own. Just faster. Of course, you need to know what is the right path to follow and how to guide it
in the right direction. But that is what I'm doing as tech lead all the time anyway. The turnaround times are just reduced a lot, AI
can now generate the amount working code within hours which would previously keep a developer busy for weeks. The initial version of
this page has an ui framework and internationalization integrated, unit and ui testing, static code analysis and a deployment all set
up and running.

From the first idea to writing my first actual blog post it was less than 10 hours of work. I guess if I had done
the research and code writing the conventional way, it would have taken me something like 50-60 hours. And my guts say
this estimation is optimistic.

So what will happen next? In the first place the 10 hours will become less. The AI still struggles to understand the
structure of a project as a whole. Even when I repeatedly point to the initially created architecture document, it tends
to become creative and do something different. Just like real human developers. It is obvious that it struggles with
complexity which involves many layers of abstraction. That is in the end the time-consuming part, code is generated
in no time. Often it is already working correctly in the current scope which is seen, but it still needs a lot of my
attention to repeat the same things over and over again to keep it on track. And the more you are using it, you
feel that it is not learning. It keeps the context of the current chat but in the end you have to tell it exactly the
same over and over again. That is the difference to a matured engineering team or my 3 years old daughter.
They memorize and train dynamically. Nonetheless, if AI quality proceeds to improve with the speed it currently does
for just a few more years it will make large scale software projects possible which are driven by just a single person.
